PageHelper 原理

先来看一下分页插件的简单用法：

对于插件机制我们上面已经介绍过了，在这里我们自然的会想到其所涉及的核心类 ：PageInterceptor。

拦截的是Executor 的两个query()方法，要实现分页插件的功能，肯定是要对我们写的sql进行改写，那么一定是在 intercept 方法中进行操作的，可以发现调用dialect的方法进行判断需要执行的逻辑，正常分页查询需要执行ExecutorUtil.pageQuery

首先使用dialect.processParameterObject设置分页参数

,再使用dialect.getPageSql返回方言获取分页 sql，然后处理额外传入的参数

本质上就是根据不同的数据库方言来生成对应的分页和排序sql，然后进行拼接。分页数据位置com.github.pagehelper.page.PageMethod#LOCAL_PAGE，是一个ThreadLocal变量

特点

  1. 数据库方言，针对不同数据库进行实现

  2. countCache：缓存查询分页的MappedStatement,位于com.github.pagehelper.cache包下，默认有GuavaCache（google guava）和SimpleCache（Mybatis自身）两种方式，第三方接入的话需要继承com.github.pagehelper.cache.Cache接口
